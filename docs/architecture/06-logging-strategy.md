# 1. Purpose

This document describes the logging strategy for the BookSmart platform. It defines log levels, standardized formats, context tracing (MDC), and guidelines for logging implementation.

---

# 2. Log Implementation Frameworks

*   **API**: Simple Logging Facade for Java (**SLF4J**).
*   **Engine**: **Logback** (bundled with Spring Boot Starter Web).

---

# 3. Log Levels

BookSmart enforces standard log levels to facilitate parsing and filtering in production:

*   **ERROR**: System faults, database connection losses, payment gateway failures, and exceptions that halt requests or require immediate support team attention.
*   **WARN**: Unexpected events that do not halt execution but require attention (e.g., slow API responses, validation errors, invalid logins).
*   **INFO**: Standard operational events (e.g., successful database migrations, application startup, booking creation events, successful payments).
*   **DEBUG**: Granular developer diagnostics (e.g., internal algorithm steps, SQL query parameter details, security filter checks).
*   **TRACE**: Maximum detail, typically left disabled except during active debugging.

---

# 4. Structured Logging & Context Tracing (MDC)

To trace requests across layers, the system uses Logback's **MDC (Mapped Diagnostic Context)**.

### Mapped Variables
*   `trace_id`: A unique UUID generated by a servlet filter at the entry point of each HTTP request. Passed in logs throughout the execution thread.
*   `user_id`: The ID of the authenticated user making the request (if authenticated).

### Console Log Pattern
```xml
%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - [TraceID: %X{trace_id}] [UserID: %X{user_id}] - %msg%n
```

### JSON Format in Production
For production environments, Logback logs in raw single-line JSON format (using `LogstashEncoder`) to allow easy ingestion into search indexes (ELK, Datadog, or Grafana Loki).

---

# 5. Security & Sensitive Data Masking

Logs must not expose personal or security-sensitive data.

### Strictly Prohibited Log Targets
*   Plaintext user passwords.
*   OAuth refresh tokens and JWT access tokens.
*   Full credit card numbers or payment credentials.
*   Customer identification details (unless hashed or masked).

### Implementation
A custom Logback `PatternLayout` or logging filter is implemented to replace sensitive patterns with `[MASKED]` (e.g., regex checks looking for `"password": "..."`).
