# 1. Base URL

All APIs in the BookSmart application share the following base paths:

*   **Local Development**: `http://localhost:8080/api/v1`
*   **Staging Environment**: `https://staging.booksmart.vn/api/v1`
*   **Production Environment**: `https://api.booksmart.vn/api/v1`

---

# 2. Protocol Constraints

*   **HTTP Protocol**: Enforced **HTTPS** in Staging and Production. Non-secure requests are redirected to HTTPS.
*   **Media Type**: Request and response payloads must utilize **JSON** (`Content-Type: application/json`).

---

# 3. HTTP Methods

We adhere to RESTful standard HTTP methods:

*   `GET`: Read resource details or retrieve collection lists.
*   `POST`: Create new resources or execute actions (e.g. login, check-in).
*   `PUT`: Update existing resources in full.
*   `PATCH`: Modify specific fields of a resource.
*   `DELETE`: Soft-delete or physically remove resources.

---

# 4. Standard Headers

### Request Headers
*   `Content-Type`: `application/json` (Required for POST, PUT, PATCH).
*   `Accept`: `application/json`.
*   `Authorization`: `Bearer <Access_Token>` (Required for all protected endpoints).

### Response Headers
*   `Content-Type`: `application/json;charset=UTF-8`.
*   `Cache-Control`: `no-store, no-cache, must-revalidate` (for sensitive endpoints).
*   `X-Trace-ID`: Unique correlation UUID generated by the gateway/app for logs tracking.

---

# 5. Rate Limiting

To prevent API abuse and DDOS attacks:
*   Standard endpoints: **100 requests per minute** per user IP address.
*   Authentication endpoints (login/register): **10 requests per minute** per user IP address.
*   Failed rate limit requests return `HTTP 429 Too Many Requests`.
